---
title: "Get Product Details"
description: "Retrieve detailed information about a specific product including variants and reviews"
---

## Endpoint

<CodeGroup>
```bash GET
GET /api/v1/products/{id}/
```
</CodeGroup>

## Authentication

This endpoint does not require authentication. It is publicly accessible.

## Path Parameters

<ParamField path="id" type="integer" required>
  The unique identifier of the product
</ParamField>

## Response

<ResponseField name="id" type="integer">
  Unique product identifier
</ResponseField>

<ResponseField name="name" type="string">
  Product name
</ResponseField>

<ResponseField name="category" type="string">
  Category name the product belongs to
</ResponseField>

<ResponseField name="description" type="string">
  Detailed product description
</ResponseField>

<ResponseField name="store" type="string">
  Hyperlink to the store details
</ResponseField>

<ResponseField name="base_price" type="string">
  Base price of the product (decimal format)
</ResponseField>

<ResponseField name="shipping_fee" type="string">
  Shipping fee for the product (can be null)
</ResponseField>

<ResponseField name="rating" type="float">
  Average customer rating calculated from all reviews (null if no reviews)
</ResponseField>

<ResponseField name="is_standalone" type="boolean">
  Whether the product has variants or is a standalone item
</ResponseField>

<ResponseField name="total_stock_level" type="integer">
  Total available stock quantity across all variants
</ResponseField>

<ResponseField name="total_sold" type="integer">
  Total number of units sold across all variants
</ResponseField>

<ResponseField name="is_available" type="boolean">
  Current availability status (true if stock_level > 0)
</ResponseField>

<ResponseField name="is_active" type="boolean">
  Whether the product is active in the system
</ResponseField>

<ResponseField name="created" type="string">
  ISO 8601 timestamp of product creation
</ResponseField>

<ResponseField name="updated" type="string">
  ISO 8601 timestamp of last update
</ResponseField>

<ResponseField name="media" type="array">

  <Expandable title="Media Object">

      URL to the media file
    </ResponseField>
    
    <ResponseField name="is_primary" type="boolean">
      Whether this is the primary product image
    </ResponseField>
    
    <ResponseField name="file_size" type="float">
      File size in megabytes
    </ResponseField>
    
    <ResponseField name="file_type" type="string">
      Type of media: `image`, `video`, or `unknown`
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="product_offers" type="array">
  Array of active offers applicable to this product
</ResponseField>

<ResponseField name="variants" type="array">
  Array of product variants with different attributes and pricing

    <ResponseField name="is_default" type="boolean">
      Whether this is the default variant for the product
    </ResponseField>
    
    <ResponseField name="price_adjustment" type="string">
      Amount to add to the product's base price (can be negative)
    </ResponseField>
    
    <ResponseField name="stock_level" type="integer">
      Available stock quantity for this variant
    </ResponseField>
    
    <ResponseField name="quantity_sold" type="integer">
      Number of units sold for this variant
    </ResponseField>
    
    <ResponseField name="is_active" type="boolean">
      Whether this variant is active
    </ResponseField>
    
    <ResponseField name="created" type="string">
      ISO 8601 timestamp of variant creation
    </ResponseField>
    
    <ResponseField name="updated" type="string">
      ISO 8601 timestamp of last update
    </ResponseField>
    
    <ResponseField name="attributes" type="array">
      Array of variant attributes (e.g., color, size)
      
      <Expandable title="Attribute Object">
        <ResponseField name="attribute" type="string">
          Attribute name (e.g., "Color", "Size")
        </ResponseField>
        
        <ResponseField name="value" type="string">
          Attribute value (e.g., "Red", "Large")
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="reviews" type="array">
  Array of customer reviews for this product
  
  <Expandable title="Review Object">
    <ResponseField name="id" type="integer">
      Review ID
    </ResponseField>
    
    <ResponseField name="user" type="string">
      Username of the reviewer
    </ResponseField>
    
    <ResponseField name="review_text" type="string">
      Review content/comments
    </ResponseField>
    
    <ResponseField name="rating" type="integer">
      Rating value (1-5):
      - 1: Very Bad
      - 2: Unsatisfied
      - 3: Just There
      - 4: Satisfied
      - 5: Very Satisfied
    </ResponseField>
    
    <ResponseField name="sentiment" type="string">
      Analyzed sentiment: `POSITIVE`, `Neutral`, or `Negative`
    </ResponseField>
    
    <ResponseField name="sentiment_score" type="float">
      Numerical sentiment score (can be null)
    </ResponseField>
    
    <ResponseField name="is_active" type="boolean">
      Whether the review is active/visible
    </ResponseField>
    
    <ResponseField name="created" type="string">
      ISO 8601 timestamp of review creation

    <ResponseField name="updated" type="string">
      ISO 8601 timestamp of last update
    </ResponseField>
    
    <ResponseField name="reviewimage_set" type="array">
      Array of images attached to the review
      
      <Expandable title="Review Image Object">
        <ResponseField name="id" type="integer">
          Image ID
        </ResponseField>
        
        <ResponseField name="image" type="string">
          URL to the review image
        </ResponseField>
        
        <ResponseField name="is_primary" type="boolean">
          Whether this is the primary review image
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

## Request Example

<CodeGroup>
```bash cURL
curl -X GET "https://api.example.com/api/v1/products/1/" \
  -H "Accept: application/json"
```

```python Python
import requests

url = "https://api.example.com/api/v1/products/1/"
response = requests.get(url)
product = response.json()
print(f"Product: {product['name']}")
print(f"Price: ${product['base_price']}")
```

```javascript JavaScript
fetch('https://api.example.com/api/v1/products/1/')
  .then(response => response.json())
  .then(product => {
    console.log(`Product: ${product.name}`);
    console.log(`Price: $${product.base_price}`);
  });
```
</CodeGroup>

## Error Responses

<ResponseField name="404" type="Not Found">
  Product with the specified ID does not exist
  
  ```json
  {
    "detail": "Not found."
  }
  ```
</ResponseField>

## Notes

- Only active products are returned (products where `is_active=True`)
- The rating is automatically calculated from all product reviews
- Stock levels and availability are automatically updated when variants change
- The `price_adjustment` on variants is added to the `base_price` to get the actual variant price
- Standalone products (is_standalone=True) can only have one default variant
- Review sentiment analysis may be automatically performed on review submission
